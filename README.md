# Droplet Tracker

Обработка видео с каплями воды (распознавание, отслеживание, определение скорости и угла движения).

## О проекте

**Предметная область:** исследование поведения (разбрызгивания) капель жидкости на поверхностях.

**Цель проекта:** автоматическое детектирование и отслеживание капель на видео, вычисление кинематических характеристик (скорость, угол движения, изменение площади).

**Сценарий работы:**
1. Пользователь имеет видеофайл (`.mp4`) с каплями на контрастном фоне.
2. Запускает Docker-контейнер с указанием входного видео, путей для выходного видео и CSV, а также параметров трекинга.
3. Скрипт покадрово обрабатывает видео:
   - Бинаризация и поиск контуров.
   - Фильтрация по площади.
   - Отслеживание капель между кадрами (по положению, площади, размерам).
   - Вычисление скорости и угла движения для каждой капли.
4. Результаты:
   - **Выходное видео** — исходные кадры с нарисованными прямоугольниками вокруг капель.
   - **CSV-файл** — таблица с записями для каждой капли в кадре (Area, Width, Height, Center X, Center Y, Angle, Speed).

## Структура проекта

```bash
├── app/
│ ├── init.py # делает app пакетом Python
│ ├── cli.py # парсинг аргументов командной строки
│ ├── process_data.py # логика обработки кадров и трекинга
│ └── main.py # точка входа
├── tests/
│ └── test_process_data.py # юнит-тесты
├── Dockerfile
├── docker-compose.yaml # продакшен и dev-сервис
├── docker-compose.test.yaml # среда для линтеров и тестов
├── requirements.txt # продакшен-зависимости
├── requirements-dev.txt # зависимости для разработки
├── .env.example # пример переменных окружения
├── .gitignore
└── README.md
```

### Ключевые модули

- **`cli.py`** – обработка аргументов командной строки (функция `parse_args`).
- **`process_data.py`** – содержит:
  - `process_frame()` – поиск капель на одном кадре.
  - `is_same_droplet()` – сравнение капель между кадрами.
  - `process_video()` – основной цикл обработки видео.
- **`main.py`** – точка входа: парсит аргументы и вызывает `process_video`.

## Разработка и тестирование

### Запуск линтеров и тестов

Проект использует `isort` (сортировка импортов), `flake8` (проверка стиля) и `pytest` (юнит-тесты). Все проверки запускаются в изолированном контейнере:

```bash
docker-compose -f docker-compose.test.yaml run --rm lint-and-test
```

Команда выполняет:

- `isort --check-only --diff` – проверка порядка импортов

- `flake8` – проверка соответствия PEP8

- `pytest --cov=app ` – запуск тестов с оценкой покрытия кода

## Настройка, сборка и запуск

### Настройка путей в `.env`

```env
INPUT_DIR=/путь/к/папке/с/видео
OUTPUT_DIR=/путь/к/папке/для/результатов
```

### Сборка и запуск

```bash
docker compose build
docker compose run --rm tracker input/input_video.mp4 output/output_video.avi output/data.csv
```

## Параметры

| Параметр | Значение по умолчанию | Описание |
|----------|----------------------|-----------|
| input_video | обязательный | Путь к исходному видео |
| output_video | обязательный | Путь для обработанного видео |
| data_csv | обязательный | Путь для CSV с результатами |
| `--position-threshold` | 100 | Максимальное расстояние между центрами капель (пиксели) |
| `--area-ratio` | 0.10 | Допустимое изменение площади (0.10 = 10%) |
| `--dim-ratio` | 0.10 | Допустимое изменение размеров (0.10 = 10%) |
